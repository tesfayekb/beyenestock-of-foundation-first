## 2. Spec Intent Summary (verified by GPT)

**(First-pass draft for GPT-5.5 Pro to refine.)**

AI-SPEC-008's load-bearing architectural commitment is its first paragraph: "Databento OPRA stream → deterministic flow_alpha_engine → Redis live snapshots → Supabase durable bars → Items 5/6/Governor as features. **The LLM never reads raw OPRA data.**" (Spec line 14). The engine is a **deterministic feature producer**, NOT a new decision layer; OPRA features inform decisions made by Items 5 (Volatility Engine), 6 (Meta-Labeler), and 1 (AI Risk Governor) but never directly chooses trades or modifies sizing. The technical core is **20 production features** (Spec §1 lines 22–76) computed from the Databento OPRA NBBO+trades stream via Lee-Ready aggressor inference (Spec §1 lines 33–46) yielding a `customer_delta` signed flow input; features cluster into 6 categories: premium concentration (z-scored, share-of-total), directional pressure (signed/unsigned, accelerating, persistent), put/call imbalance (dollar-weighted, aggressor-filtered), delta-bucket pressure (wing 0.05-0.12, short 0.12-0.22, medium 0.22-0.35, ATM 0.35-0.60), event detection (sweep clusters, block trades, key-strike concentration), and dealer-hedge proxies (dealer_hedge_pressure_z, flow_price_divergence). The engine emits stable features only on 1-minute and 5-minute closes (Spec §2 lines 167–168) — never per-tick — to prevent downstream model thrash; runs in a **separate Railway worker process** (Spec §4 line 247) to isolate latency risk from the trading cycle. The engine's outputs feed (a) Item 5 (Vol Fair-Value Engine — 8 features per §5 lines 270–281 for short-gamma EV penalty + realized-vol forecast confidence + long-vol confidence), (b) Item 6 (Meta-Labeler — all 20 features plus quality flags per §5 line 290 as input vector), (c) AI Risk Governor (compact 5-field JSON summary per §5 lines 305–313 — `flow_direction` / `flow_strength` / `vol_demand` / `sweeps` / `price_alignment` / `confidence`). **Critical anti-creep rule (Spec §0 line 18 + §13 line 575):** Item 6 with OPRA features must beat Item 6 without OPRA features on calibration, realized utility, and drawdown — **otherwise OPRA stays logging/research only**. V0.3 scope (Spec §7) is statistical features only; pattern ML (LSTMs, transformers on tick sequences) is deferred to V0.4+ pending 6-12 months clean feature history + 500+ trade outcomes. Authority promotion path (§11): Months 1-3 collect/replay/paper-shadow with 0% live contribution; Months 4-6 advisory / meta-labeler shadow influence with +0% to +2%; Months 7-12 production if validated, +4% to +8% base case / +8% to +12% bull case. Existing `backend_agents/flow_agent.py` (343 LOC at HEAD; brief generator) is to be **role-narrowed to LLM summarizer** per Spec §8 line 460 — but `AI_BUILD_ROADMAP.md` §3.9 line 372 says "successor-replace" — a cross-document contradiction surfaced as `C-AI-008-2`.
