## 2. Spec Intent Summary (verified by GPT)

**(First-pass draft for GPT-5.5 Pro to refine.)**

AI-SPEC-009's load-bearing architectural commitment is its `## Architectural Commitment` opening (Spec line 14): "The exit optimizer is a deterministic adaptive layer that recommends EARLIER exits than static rules when forward EV signals justify it. It NEVER: Causes later exits than static rules / Overrides constitutional gates (14:30 close, -3% halt, hard stops) / Rolls or reopens positions / Operates on every tick (would create noise-driven exits)." The engine is a **deterministic forward-EV adapter** that REUSES Item 5's distribution machinery as a current-state forward-EV computer (Spec §1 line 28: "Reuse Item 5's distribution machinery as a current-state forward-EV adapter. Do NOT build a separate vol model"); it CONSUMES Item 8 OPRA flow features as conservative distribution modifiers (Spec §1 lines 80–98); it ADVISES in V0.3 and EARNS limited authority in V0.4 strategy-by-strategy after rigorous validation (Spec §11). The technical core is **two-side EV computation** (Spec §1 lines 41–71 — `EV_exit_now` from current close-cost minus adaptive slippage; `EV_hold` from forward-path simulation with static-exit policy applied inside each path) yielding `exit_advantage = EV_exit_now - EV_hold` and `exit_advantage_R = exit_advantage / max_risk` decision variables; **trajectory features** (Spec §2 lines 102–134 — `giveback_R`, `pnl_path_vol`, `profit_velocity`, multiplicative `EV_uncertainty_R_adjusted`) modify confidence in the EV computation NOT raw EV value (Spec §2 lines 136–139 — "This prevents the optimizer from becoming a hidden trailing stop"); **hybrid evaluation cadence** (Spec §3 — 120s normal / 30–60s stress with 7 enumerated state-change triggers); **strategy-specific exit logic** (Spec §6 — IC, IB, PCS, Long Straddle, Debit Call Spread; **5 of 10 canonical strategies** per `B-AI-009-9`); **adaptive exit slippage formula** (Spec §7 — `adjustment_factor` capped at 3x with vol stress + time pressure + spread widening + stress flag terms); **constitutional gate interaction** (Spec §8 — 14:30 mandatory close embedded in every simulated path; rejection if computed EV depends on post-14:30 continuation). Authority promotion path (Spec §11): V0.3 advisory only (logs recommendations; static rules execute exits); V0.4 limited production authority requires **all 11 enumerated promotion gates** (200+ trades; 40 per strategy; 60+ paper-shadow sessions; positive `adaptive_exit_value` over rolling 60 sessions; realized P&L improvement > 5% versus static; max drawdown not worse; worst day not worse; profit factor not worse; slippage < 30% of improvement; `Wilson_lower_95(useful_exit_rate) > 0.30`; `adaptive_exit_value > 0`); even at V0.4, the optimizer can ONLY close/reduce — never roll, reopen, delay static exits, or override constitutional gates (Spec §11 lines 622–628). **Critical anti-creep rule (Spec §0 line 22 + §11 line 630):** "A bad entry filter skips one trade. A bad exit optimizer can damage every open trade. Promotion to production authority must reflect this asymmetry." Existing rule-based exit logic at `backend/position_monitor.py` (858 LOC at HEAD; 11 distinct exit_reasons emitted per V9) remains the **Layer 1 static-exit safety net** per Spec §16 line 779 ("Static exits remain the execution backbone; adaptive exits become an earlier-risk-reduction overlay only after proving positive value strategy-by-strategy") — this Layer 1 + Layer 2 COEXISTENCE is surfaced as `C-AI-009-2` for explicit operator ratification of the cutover policy.
