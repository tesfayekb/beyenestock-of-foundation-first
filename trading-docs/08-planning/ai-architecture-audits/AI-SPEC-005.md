## 2. Spec Intent Summary (verified by GPT)

**(First-pass draft for GPT-5.5 Pro to refine.)**

AI-SPEC-005's load-bearing architectural commitment is its **first paragraph**: "The Volatility Fair-Value Engine is a **deterministic pricing/forecasting module, not an AI module**" (Spec §0 line 13). The engine's authority chain is `Vol Engine → Meta-labeler + AI Risk Governor → Deterministic Arbiter → Execution` (Spec §0 line 15) — the engine produces measurements (`rv_forecast_final`, `iv_rv_ratio`, `skew_z`, surface scores, `strategy_ev_table` top-3) and never directly chooses trades. The LLM never reads raw option chains or performs vol math; it consumes only a compact summary (Spec §0 line 17, Spec §9 lines 675–687). The technical core is a **HAR-RV remainder model** (Spec §1 lines 23–38) that forecasts log of **remaining realized variance** from decision time to planned exit — not generic daily volatility — using a 7-coefficient regression on prior-session / prior-week / prior-month log-RV plus intraday running RV plus `overnight_gap`, `vix_change_from_prior_close`, `event_day_flag`, `time_bucket`. Critical anti-double-counting rule (Spec §1 line 187–190): `skew_z` is the production decision signal; `smile_asymmetry_score` / `smile_width_score` / `tail_width_score` are meta-labeler features ONLY. The engine's outputs feed (a) Item 1 (Risk Governor) Card features per AI-SPEC-001 §3 LLM Output, (b) Item 6 (Meta-Labeler — future audit P1.3.6) feature vector, (c) the strategy-conditional EV table per Spec §4 lines 309–438 with strategy-specific decision rules for IC / IB / PCS / Long Straddle / DCS, and (d) the Item 4 (Replay Harness) calibration validation per Spec §7 lines 543–639 with promotion gates (Baseline E must beat Baseline F — full stack vs full-stack-minus-Vol-Engine). **The spec is silent on D-016** (Volatility Blending: `sigma = max(realized, 0.70 × implied)`) despite producing the canonical realized-vol input — this is the audit's most consequential finding (`C-AI-005-1`). Authority promotion path: Months 1–3 advisory-only / paper-only, Months 4–6 short-gamma filter only with limited authority, Months 7–12 mature if validation passes (Spec §10 lines 695–702); long-vol live authority deferred to V0.3+ pending ≥30 clean signals. Pre-V0.2 data requirements (Spec §11 lines 718–735) state 6 months of **post-Commit-4 data** is enough for V0.2 ship with constrained authority — and 60+ clean signals required for unconstrained long-vol authority — making this spec a downstream consumer of both AI-SPEC-002 (calibration labels) and AI-SPEC-010 (counterfactual training accessors with `calibration_eligible = true` filtering).
